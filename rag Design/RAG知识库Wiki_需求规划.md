## 1. 项目背景

团队/部门的本地文件夹中积累了大量知识资产(需求文档、设计规范、会议纪要、技术规范等),这些资料分散、检索困难,知识复用高度依赖"找对人"。本项目旨在用 **RAG(检索增强生成)** 技术自动整理本地知识库,生成可由团队浏览与检索的**网站内部 Wiki**,降低知识查找成本、加速新人 onboarding。

---

## 2. 目标用户与痛点

| 角色 | 目标用户 | 核心痛点 | 典型场景 |
| --- | --- | --- | --- |
| 团队成员 | 研发 / 产品 / 运营 | 资料散落、靠人肉翻找、检索低效 | 查找历史技术决策、需求背景、规范 |
| 新成员 | 入职不久的员工 | 不了解知识分布、onboarding 慢 | 通过 Wiki 快速建立领域认知 |
| 知识管理者 | 技术负责人 / 文档 owner | 知识随人走、难以沉淀 | 统一沉淀团队规范与经验 |

---

## 3. 业务目标

1. 将本地知识库**自动转换为结构化、可浏览的内部 Wiki**
2. 支持**语义检索**与(可选)**AI 问答**,相比文件名搜索显著提升命中率
3. 降低知识查找时间,提升知识复用率与新人上手速度
4. (可量化指标,待确认)如:平均查找耗时下降 X%、Wiki 页面覆盖率 Y%

---

## 4. 功能范围(建议 MVP)

| 模块 | 功能 | 优先级 |
| --- | --- | --- |
| 知识库加载 | 遍历本地文件夹,识别并加载多格式文档 | P0 |
| 文档解析 | 解析 md / docx / pdf / txt 等格式为纯文本 | P0 |
| 切片与向量化 | 文本切片 + 嵌入模型向量化 + 写入向量库 | P0 |
| 检索与生成 | 向量检索 + (重排) + LLM 生成(RAG pipeline) | P0 |
| Wiki 生成 | 将知识整理为结构化内部 Wiki 页面(目录/导航/正文) | P1 |
| 搜索/问答界面 | 网站内全文/语义搜索与 AI 问答交互 | P1 |
| 增量更新 | 监听文件夹变化,增量重建索引 | P2 |
| 权限与多用户 | 访问鉴权、多用户(如需要) | P2(已确认当前不需要) |

---

## 5. 技术架构(已确认推荐)

> 基于已确认约束:**文档为主、本地单机运行**。模型方案与 Wiki 形态由产品推荐。

| 层 | 推荐方案 | 备注 |
| --- | --- | --- |
| 开发语言 | Python 3.11 | 生态成熟 |
| RAG 框架 | LangChain + langchain-chroma | 主流、文档全 |
| 向量库 | Chroma(本地持久化) | 零运维,契合单机 |
| 嵌入模型 | 默认本地 `BAAI/bge-small-zh-v1.5`(离线);预留云端 API 开关 | 中文友好、可离线 |
| 生成模型(LLM) | 可插拔:云端 API(gpt-4o-mini 等,效果更好)/ 本地开源(Ollama+qwen2.5,离线) | 由配置切换 |
| 前端/站点 | MVP 先做静态可浏览站点 + 语义检索;问答作为可叠加模块 | 单机零运维、最快见效 |
