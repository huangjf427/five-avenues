# FiveAvenues 文旅网站 — 需求文档

> 最后更新：2026-07-20  
> 状态：V1.0 已上线；**V1.1.0（FR-8 外部知识库接入）已实现**；**V1.2（FR-9~FR-13 RAG 增强 + 社区运营增强，已实现）**——详见 `docs/PRD_V1.2_RAG社区增强.md`、`docs/V1.2_研发任务拆解.md`

## 产品定位

为天津五大道景区游客提供 **个性化行程生成** 与 **社区评价分享** 的一站式文旅指南网站。

|---|---|
| 编号 | FR-1 |
| 优先级 | P0（核心功能） |

## 非功能需求

### NFR-1 零依赖

系统不使用任何 npm 第三方包，仅依赖 Node.js 内置模块（http, fs, crypto, path, url）。

### NFR-2 匿名可选

仅登录用户可发表评价；登录用户可勾选匿名发布（默认实名），以降低身份暴露顾虑。该约束依据 PO 决策 3216ed6 / REG-14 收紧，原「游客无需注册即可发表」不再适用。

### NFR-3 审核先行

所有评价必须经管理员审核通过后才对外展示，防止不当内容。

### NFR-4 数据安全

- 密码以 scrypt 加盐哈希存储，不存明文
- 会话 token 存储在 HttpOnly Cookie 中
- `users.json` 和 `reviews.json` 已加入 `.gitignore`

### NFR-5 响应式

界面适配桌面和移动端（网格布局在小屏自动折叠为单列）。

### NFR-6 可打印

支持浏览器打印 / 导出 PDF，打印时自动隐藏表单和导航控件。

### NFR-7 知识库配置外置

WuDaDao 知识库为独立项目，其位置**不得硬编码于代码**；必须通过 `.env` 配置链接（见 FR-8）。缺省 / 不可达时应明确降级，不静默失败。

### NFR-8 埋点合规

- 埋点仅上报计数 / 标签 / 原因分类与长度，**不采集密码明文与评价正文全文**；匿名 / 游客不关联用户标识。
- 上报失败不影响主流程；沿用 NFR-1 零依赖，走 Node 内置 `fetch`（上报端点待技术负责人确认，见 PRD 第 9 节 A1）。
