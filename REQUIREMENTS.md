# FiveAvenues 文旅网站 — 需求文档

> 最后更新：2026-07-14  
> 状态：V1.0 已上线

## 产品定位

为天津五大道景区游客提供 **个性化行程生成** 与 **社区评价分享** 的一站式文旅指南网站。

---

## 功能需求

### FR-1 个性化行程生成

| 项目 | 描述 |
|---|---|
| 编号 | FR-1 |
| 优先级 | P0（核心功能） |
| 用户故事 | 作为游客，我希望根据我的旅行目的和兴趣生成定制化的五大道行程 |
| 输入 | 旅行目的（单选）、兴趣方向（多选，最多 5 项）、出发日期、返程日期 |
| 输出 | 逐日行程（含景点/店铺/活动）、历史人物、建筑风貌、美食购物推荐、节事日历、季节提示、资料来源 |
| 约束 | ① 基于本地 WuDaDao 知识库生成 ② 支持日期范围内的活动匹配 ③ 空结果时给出降级提示 |
| 状态 | ✅ 已实现 |

### FR-2 游客 Wiki — 评价发布

| 项目 | 描述 |
|---|---|
| 编号 | FR-2 |
| 优先级 | P0 |
| 用户故事 | 作为游客（无论是否登录），我希望分享对五大道行程的体验和感受 |
| 输入 | 评分（1-5 星）、评价对象（整体行程 / 具体店铺 / 具体活动）、标题（可选）、正文（1-2000 字）、标签（可选） |
| 约束 | ① **支持匿名发布**（未登录默认匿名；已登录可勾选匿名） ② 匿名时显示"匿名游客" ③ 提交后立即进入 pending 状态 |
| 状态 | ✅ 已实现 |

### FR-3 游客 Wiki — 评价展示

| 项目 | 描述 |
|---|---|
| 编号 | FR-3 |
| 优先级 | P0 |
| 用户故事 | 作为游客，我希望看到其他游客已通过的评价，帮助我做出行决策 |
| 展示内容 | 评分星条、标题、正文、标签、评价对象、作者名、发布日期 |
| 约束 | ① **仅展示已通过（approved）评价** ② 匿名评价显示"匿名游客" ③ 按时间倒序 ④ 支持按评价对象类型过滤 |
| 状态 | ✅ 已实现 |

### FR-4 游客 Wiki — 审核管理

| 项目 | 描述 |
|---|---|
| 编号 | FR-4 |
| 优先级 | P0 |
| 用户故事 | 作为管理员，我希望审核游客提交的评价，决定是否公开 |
| 功能 | ① 按状态筛选（待审核 / 已通过 / 已拒绝 / 全部） ② 通过 / 拒绝操作 ③ 拒绝时可填写审核意见（备注） ④ 已通过的条目可改为拒绝，反之亦然 ⑤ 审核操作记录操作人和时间 |
| 约束 | ① 仅 admin 角色可访问 ② 未登录返回 401，非 admin 返回 403 |
| 状态 | ✅ 已实现 |

### FR-5 我的评价

| 项目 | 描述 |

| 编号 | FR-5 |

## 非功能需求

### NFR-1 零依赖

系统不使用任何 npm 第三方包，仅依赖 Node.js 内置模块（http, fs, crypto, path, url）。

### NFR-2 匿名优先

游客无需注册即可发表评价，降低参与门槛。

### NFR-3 审核先行

所有评价必须经管理员审核通过后才对外展示，防止不当内容。

### NFR-4 数据安全

- 密码以 scrypt 加盐哈希存储，不存明文
- 会话 token 存储在 HttpOnly Cookie 中
- `users.json` 和 `reviews.json` 已加入 `.gitignore`

### NFR-5 响应式

界面适配桌面和移动端（网格布局在小屏自动折叠为单列）。

### NFR-6 可打印

支持浏览器打印 / 导出 PDF，打印时自动隐藏表单和导航控件。
