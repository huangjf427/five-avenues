## 阶段一：项目理解与调研 Brief

### 1.1 产品理解笔记（来源：REQUIREMENTS.md / DESIGN.md / overview-fr8.md）
- **一句话定位**：面向**天津五大道景区**游客的"个性化 AI 行程生成 + 游客 Wiki 评价社区"一站式文旅指南网站。
- **核心功能（已实现 V1.0–V1.1）**
  - FR-1 个性化行程生成（RAG 读取外部 WuDaDao 知识库，含景点/店铺/活动/历史/建筑/美食/节事/季节提示/来源）。
  - FR-2~FR-5 游客 Wiki：评价发布（登录+可匿名）、评价展示（仅 approved）、审核管理（admin）、我的评价。
  - FR-6 账号管理（scrypt 加盐哈希、7 天会话）。
  - FR-7 知识库手动同步。
  - FR-8 外部知识库接入（`.env` 配置 `WUDADAO_KB_PATH`，RAG 读取，不可达降级）。
- **规划中 V1.2**：FR-9 RAG 配置引导/缺省体验、FR-10 行程溯源展示、FR-11 评价"有用"投票、FR-12 评价举报、FR-13 审核工作台增强、NFR-8 埋点合规。
- **非功能特征（关键差异点）**：NFR-1 **零 npm 第三方依赖**（仅 Node.js 内置模块）、NFR-2 匿名可选、NFR-3 审核先行、NFR-4 数据安全（scrypt/HttpOnly Cookie）、NFR-5 响应式、NFR-6 可打印、NFR-7 知识库配置外置、NFR-8 埋点合规。
- **技术栈**：Node.js 原生 HTTP（无框架）、内存会话、JSON 文件存储、RAG 用 Node 全局 `fetch` 调智谱/DeepSeek/OpenAI 兼容 API，向量检索为纯 JS 余弦相似度。
- **商业模式（现状）**：**免费、无交易闭环**（不含门票/餐饮预订），面向游客的信息工具。

### 2.3 各竞品画像摘要（关键事实 + 来源）

**① 大众点评 / 美团**
- 点评：MAU 约 1.55 亿（美团 2026-03 披露），2025 年评价近 4.5 亿条；有评分/UGC/导航，**无 AI 行程**，重交易。
- 美团：年交易用户 7.7 亿、活跃商户 1450 万（2024 年报）；已推 LongCat 大模型与"小美"AI Agent，交易闭环最强但**隐私收集最多**。
- 关系：间接替代 + 互补。FA 应避开正面竞争，以"本地深度 + RAG 个性化 + 隐私"差异化，把点评当互补层。

**② 小红书**
- 国内 MAU 2.42 亿（2025-05）；旅行兴趣月活 2.3 亿、月均旅行搜索 16 亿次。是攻略灵感阵地，**无评分/导航/AI 行程**。
- 关系：间接替代（种草）+ 互补。FA 的"可信行程 + 匿名评价"可承接小红书种草后的"落地决策"。

**③ 马蜂窝**
- MAU 约 550–630 万（2025-10，较疫情期更低，数值存疑）；2025 年上线 **AI 行程规划**（旅游垂直模型 + 自有 15 年数据 + RAG），含游记/点评/酒店门票交易闭环。
- 关系：间接替代 + 互补。对五大道类小景区本地历史深度有限、商业化重、非隐私友好、非自托管。

**④ 高德地图 / 百度地图**
- 高德：MAU 近 9 亿（2024.10–2025.9，QuestMobile）；2025 版"小高老师"AI 智能体覆盖行前-行中-行后全旅程行程规划；2025Q3 首次盈利。
- 百度：MAU 6 亿、POI 3.4 亿（2025）；AI 向导（文心）用户破亿、日均对话超千万（2025-03）。
- 关系：间接替代（导航/吃什么）+ 互补。二者均有"行程规划 + POI + 本地生活"，但缺五大道历史深度、位置隐私敏感、非自托管。

**⑤ 携程**
- AI 行程助手 2025-09-16 双端上线，集成"规划—调整—预订—订单"全流程，含门票/酒店/交通闭环与数亿 UGC；2025 净营收 624 亿（+17%）、GMV 约 1.1 万亿、国内 OTA 市占约 56%。
- 依托 2000 万 POI + 实时数据 + 数亿评价，但**五大道本地历史深度弱**（偏交易与口碑）。
- 关系：替代"行程+订票"，不替代"历史知识 + 匿名 Wiki 社区"。

**⑥ 通用大模型（豆包等）**
- 豆包 MAU 2.3 亿（2025Q4，QuestMobile 2026-02）；豆包/文心/智谱/Kimi/ChatGPT 均支持中文旅行规划与联网搜索，免费 + 会员增值。
- 弱项：**五大道本地历史深度弱**，易泛泛而谈甚至编造（幻觉风险）；无本地 RAG、无审核 UGC、无自托管、对话数据被收集。
- 关系：弱替代"行程怎么排"，完全不替代"可信本地历史 + RAG + 匿名 Wiki 社区"。

**⑦ TripAdvisor（猫途鹰）**
- 月访问约 9770 万（Similarweb 2026-01），>10 亿条点评；AI 行程生成器 2023-07 上线（基于 OpenAI）。
- 但对**中国小景点覆盖浅**：五大道仅 436 条点评（4.2/5），多停留在 2019–2020 年；本地化弱、无中文深度互动。
- 关系：国际对标（AI 行程 + RAG 思路相似），FA 聚焦单一街区 + 官方知识库 + 匿名本地 UGC 形成细分壁垒。

**⑧ 五大道官方渠道**
- "云上五大道"微信小程序：AR 导览、建筑二维码扫 3D 全景与历史渊源（和平区政府 2026-07-14 答复）；游客中心 2025-12 升级（VR/AR、"时光听筒"）；AI 虚拟导游"和小棠"+ 四足机器狗巡游（2025 海棠花节）；抖音话题播放超 4.1 亿次。
- 政府公益免费，权威可信，但**无个性化 AI 行程、无 UGC 评价社区、无匿名互动**。
- 关系：**互补/背书**而非替代。FA 可引用官方史料作 RAG 知识库，并填补"个性化行程 + UGC 社区"空白。

### 2.4 来源可信度评估
- **高可信**：上市公司财报/年报（美团 2024 年报、携程 2025 财报）、官方披露（美团 MAU、Similarweb 估算注明）。
- **中可信**：媒体报道（QuestMobile、各家新闻稿）、政府答复（和平区政府）。
- **存疑/局限**：马蜂窝 MAU 多口径矛盾（取较近"数百万人"并标注）；官方小程序实际可用性未公开验证；部分大模型 MAU 未单独披露（Kimi/智谱）。凡存疑项已在画像中标注"未找到可靠公开数据"或"存疑"。

### 3.3 能力差异与体验亮点
- **FA 独有组合**：垂直街区 RAG 个性化行程 + 审核制匿名 UGC + 零依赖可自托管 + 隐私最小化，这一组合在所有竞品中**唯一**。
- **体验亮点**：行程可打印/导出（NFR-6）；RAG 溯源（FR-10 规划）可显性化"结论依据什么"，建立信任；匿名降低评价顾虑。

### 3.4 风险点（FA 自身）
- 无地图/导航 → 行中体验断层；无交易 → 变现弱。
- UGC 冷启动：新站评价少，对比点评/小红书海量内容，迁移成本高。
- 单景区天花板；AI 依赖外部 `RAG_API_KEY`，未配则降级规则，深度受知识库质量约束。
- 品牌/流量为零，获客难。

### 3.5 竞品背后的产品策略提炼
- **点评/美团/携程**："交易闭环 + 数据飞轮"——用 UGC/POI 引流，交易变现，AI 是提效与留存手段，非差异核心。
- **地图（高德/百度）**："LBS 入口 + 全旅程 AI 化"——把导航/POI 变成行程与本地生活入口。
- **小红书**："内容种草 + 社区信任"——靠 UGC 情绪价值，不做交易闭环（导流）。
- **官方**："权威背书 + 沉浸体验（AR/VR）"——重形象与公共服务，不做个性化与互动。
- **FA 可切入的空白**：巨头赌"广度+交易"，官方赌"权威+体验"，**无人赌"垂直可信本地知识 + 隐私友好 + 可自托管"**——这正是 FA 的位。

## 阶段四：洞察分析（事实 / 判断 / 建议分离）

### 4.1 SWOT
| | 内容 |
|---|---|
| **Strengths（优势）** | ① 垂直街区文史知识库（WuDaDao）RAG，巨头难复制的深度；② 隐私友好（零依赖/scrypt/匿名/埋点合规）；③ 可自托管、零部署成本，适合作/政/景区合作；④ RAG 溯源 + 规则降级，避免大模型幻觉；⑤ 审核先行 + 匿名，UGC 风险可控。 |
| **Weaknesses（劣势）** | ① 无交易闭环，变现弱；② 无地图/导航，行中断层；③ UGC 冷启动；④ 单景区天花板；⑤ AI 依赖外部 API；⑥ 零品牌流量，获客难。 |
| **Opportunities（机会）** | ① 官方史料接入 RAG + 争取合作背书；② V1.2 社区质量（投票/举报/审核台）建可信 UGC 壁垒；③ 隐私合规叙事（数据最小化监管趋势）；④ 文保/研学/政务自托管版（零依赖产品化）；⑤ 巨头盲区=垂直可信本地知识。 |
| **Threats（威胁）** | ① 巨头 AI 化（小高老师/AI 向导/携程 AI/豆包）免费截流普通行程规划；② 点评/小红书已满足看评价/种草；③ 官方 AR 导览可能覆盖导览需求；④ 大模型联网可临时生成五大道行程；⑤ 单景区流量有限，C 端变现难。 |

### 4.2 机会优先级（含 V1.2 契合度）
| 优先级 | 机会 | 与 V1.2 关系 | 说明 |
|---|---|---|---|
| **P0** | 官方史料接入 RAG + 合作背书 | 支撑 FR-8/FR-1 | 强化"可信深度"并解决获客 |
| **P0** | 社区质量（投票/举报/审核台） | FR-11/12/13 | 建立可信 UGC 壁垒，差异核心 |
| **P1** | RAG 溯源/引导显性化 | FR-9/10 | 把"可信生成"变成可感知卖点 |
| **P1** | 行程导出/分享到地图 | 新增建议 | 低成本的导航断层补位，与巨头互补 |
| **P2** | 文保/研学自托管版 | 放大 NFR-1 | 把零依赖能力产品化为 B2G/B2B |
| **P2** | 多景区知识库复制 | 扩展 | 突破单景区天花板 |

### 4.3 竞品差距地图（FA 应如何应对每类竞品）
- **vs 点评/小红书**：我们有本地深度行程 + 匿名可信社区；缺海量 UGC/流量 → **不硬刚，做互补 + 内容导入**。
- **vs 地图/携程**：我们有垂直文史 RAG + 隐私；缺导航/交易 → **互补（导出/嵌入）**，不自建交易。
- **vs 通用大模型**：我们有无幻觉可信 + 本地 RAG；缺广度 → **强调专业度与可溯源**。
- **vs 官方**：我们有个性化行程 + UGC；缺权威 → **接入官方作知识源 + 争取背书**。

### 4.4 用户需求洞察
- 游客要"**可信的本地专业知识**"（非泛泛而谈）→ RAG 深度是核心锚点。
- 游客要"**他人真实经验判断值不值**"→ 审核制 + 匿名的可信 UGC。
- 游客（尤其注重隐私/研学/亲子群体）要"**不被追踪**"→ 零依赖自托管是隐性差异。
- 游客要"**一站式**"→ 我们缺导航/交易是体验断层，需以互补而非自建解决。

## 阶段五：竞品分析报告（结构化输出）

### 5.1 调研背景
FiveAvenues 是天津五大道垂直文旅指南，已上线 V1.0–V1.1（AI 行程 + 游客 Wiki），V1.2 规划 RAG 增强与社区运营增强。本分析基于 `REQUIREMENTS.md` 等需求文档，明确其在"游客规划行程 + 参考他人评价"场景下的竞争格局与差异化空间。

### 5.2 调研目标
界定竞品范围、量化能力差距、提炼 FA 的护城河与机会、为 V1.2 优先级与定位/变现决策提供依据。

### 5.3 方法说明
分支 `main` 需求文档解读 + 4 集群并行联网调研（WebSearch/WebFetch，带来源与时间），统一模板结构化产出，事实/判断/建议分离。

### 5.4 核心发现
1. **竞争分三层**：UGC 点评（点评/小红书/马蜂窝）、地图+AI 行程（高德/百度/携程）、通用大模型；官方为互补方。
2. **FA 唯一组合**：垂直 RAG 文史行程 + 审核匿名 UGC + 零依赖自托管 + 隐私友好，竞品均无。
3. **巨头盲区**：赌广度+交易/权威+体验，**无人赌垂直可信本地知识 + 隐私**。
4. **最大威胁**：巨头 AI 免费截流"普通行程规划"，FA 须以"无幻觉可信 + 本地深度"守细分。
5. **官方是最佳盟友**：权威史料可作 RAG 源，且官方不做个性化行程与 UGC。

### 5.5 竞品分析（分层，详见阶段三表）
- **直接/间接替代**：点评/小红书/马蜂窝在 UGC 与社区信任占优，但缺本地文史 RAG 与隐私；FA 以深度 + 可信 + 隐私差异化。
- **地图/OTA**：高德/百度/携程在导航/交易/规模占优，FA 以垂直 RAG + 自托管互补，不自建交易。
- **通用 AI**：广度大但有幻觉与隐私问题，FA 以"可溯源可信生成"对抗。
- **国际对标**：TripAdvisor AI 行程思路相似，但对中国小景区覆盖浅，FA 凭街区级 RAG + 本地匿名 UGC 形成壁垒。
- **官方渠道**：权威 + AR，但无个性化与 UGC，是 FA 的背书与知识源合作对象。

### 5.6 机会判断
- **首选机会（P0）**：① 官方史料接入 RAG + 合作背书；② V1.2 社区质量功能（投票/举报/审核台）建立可信 UGC 壁垒。
- **次选（P1）**：RAG 溯源/引导显性化；行程导出到地图（互补而非自建导航）。
- **拓展（P2）**：文保/研学自托管版（产品化零依赖）；多景区知识库复制。

### 5.7 风险提示
- **R1 巨头 AI 截流**：高德/百度/携程/豆包免费 AI 行程，可能让"普通规划"需求不流向 FA。
- **R2 UGC 冷启动**：新站评价稀缺，社区可信壁垒短期难建立。
- **R3 单景区天花板 + C 端变现难**：需探索 B2G/B2B（自托管/定制知识库）路径。
- **R4 RAG 真实效果未验证**：需求文档注明环境未做端到端验证（密钥无效），"可信生成"需真实数据背书。

### 5.8 建议动作
**给产品团队（需求机会与功能建议）**
1. 优先落地 V1.2 社区质量（FR-11/12/13）——可信 UGC 是差异核心。
2. 做 RAG 溯源与引导（FR-9/10），把"可信"显性化。
3. 新增"行程导出/分享到高德/小红书"——低成本的导航断层补位。
4. 推动接入官方史料作 RAG 知识源（强化深度 + 背书）。

**给管理层（市场判断与决策建议）**
1. 定位"**垂直可信本地知识 + 隐私友好**"细分，不与巨头广度/交易正面对抗。
2. 官方/文保合作是获客与背书的关键路径，建议设立专项。
3. 商业模式：免费工具 + 为政企提供**自托管/定制知识库服务（B2G/B2B）**可能比 C 端广告更可行。
4. 重点防范 R1–R4，尤其用真实 RAG 评测与用户访谈验证核心假设。

## 阶段六：结论评审

### 6.1 评审议题
1. "垂直可信本地知识 + 隐私友好"定位是否成立、能否支撑获客？
2. 官方合作路径是否可行、谁是接口人？
3. 商业模式走 C 端（广告/会员）还是 B2G/B2B（自托管）？
4. V1.2 优先级是否应以"社区质量 + RAG 可信"为先？

### 6.2 关键结论
- FA 的差异化护城河 = 垂直 RAG 文史行程 + 审核匿名 UGC + 零依赖自托管 + 隐私友好，竞品均不兼具。
- 不应与巨头在"广度 + 交易"正面对抗，应走"小而深"垂直 + 互补。

### 6.3 争议点
- **是否做交易闭环？** 建议**不做**，以保持零依赖/隐私/轻量；交易可通过导出到携程/美团实现互补。
- **是否扩张多景区？** 机会存在，但需先验证单景区模型与 RAG 可复制性。

### 6.4 待确认问题
- 官方（和平区文旅/五大道管委会）合作可行性？史料授权范围？
- RAG 端到端真实效果（需有效 `RAG_API_KEY` + 真实 embedding 评测）。
- "隐私/可信"是否真为游客决策痛点（需用户访谈验证）。

### 6.5 决策项
- V1.2 优先级确认（社区质量 + RAG 可信是否 P0）。
- 是否立项探索 B2G 自托管产品。

### 6.6 行动项（可建事项分配）
1. **接触官方文旅**：调研"云上五大道"合作与史料授权 —— 负责人：BD/产品。
2. **用户访谈**：验证"隐私/可信/本地深度"是否核心痛点 —— 负责人：用研。
3. **RAG 评测**：用真实密钥跑端到端，量化行程质量与溯源准确率 —— 负责人：技术。
4. **进入需求设计**：FR-9/10/11/12/13 详细设计 —— 负责人：产品。
5. **竞品监测**：建立高德/百度/携程 AI 行程与官方渠道跟踪清单 —— 负责人：市场。
