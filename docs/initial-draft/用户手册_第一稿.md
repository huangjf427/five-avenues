## 1. 产品简介

**FiveAvenues（五大道旅行文化指南）** 是一个为**天津五大道景区游客**提供帮助的网站，主打两件事：

1. **定制行程**：根据你的旅行目的和兴趣，自动生成包含景点、店铺、活动、历史与建筑风貌、美食购物、节事日历、季节提示的逐日行程。
2. **游客 Wiki（社区评价）**：游客可以分享对五大道行程、店铺、活动的真实体验，并互相"点赞（有用）"、对不当内容"举报"，由管理员审核把关。

网站基于本地 **WuDaDao 历史知识库** 生成内容；当配置了 AI 增强（RAG）密钥时，还能给出"依据哪条知识"的溯源，让推荐更可解释。

> 技术小知识（不影响使用）：网站**零第三方依赖**，运行在 Node.js 上；知识库可放在仓库外，通过配置指向。

## 2. 快速开始

### 2.1 访问网站
在浏览器打开网站地址（默认本地部署为 `http://localhost:3000`；具体域名由部署方提供）。

### 2.2 注册账号
1. 点击页面右上角「登录 / 注册」入口，打开登录弹窗。
2. 切换到「注册」，填写：
   - **用户名**：2~24 个字符。
   - **密码**：至少 6 位。
3. 提交后**自动登录**。

> ⚠️**待核对**：注册入口的具体按钮文案与弹窗布局以线上为准（依据 `DESIGN.md` §6.3 登录/注册弹窗支持 ESC / 点遮罩 / × 关闭）。

### 2.3 登录与登出
- **登录**：在登录弹窗输入用户名密码，成功后服务端通过 Cookie（`fa_session`）保持会话，**有效期 7 天**。
- **登出**：点击登出入口清除会话。
- 查看当前用户：页面会显示当前登录用户信息（`/api/auth/me`）。

> 会话为**进程内内存态**：若服务重启，需要重新登录（`DESIGN.md` §8.3）。

### 2.4 你需要登录吗？
| 你想做的事 | 需要登录吗 |
| --- | --- |
| 浏览行程、看公开评价 | 不需要（游客即可） |
| 发表评价 | **需要**（登录后可选择匿名） |
| 对评价点「有用」、举报 | **需要** |
| 查看「我的评价」 | **需要** |
| 进入审核后台 | **需要，且须为管理员** |

> 依据：`NFR-2` 与 `FR-2`——依据 PO 决策，**未登录不可发表评价**（原"游客免注册可发"已不再适用）。

## 3. 功能详解

### 3.1 定制行程（FR-1 / FR-8 / FR-9 / FR-10）

**入口**：顶部「定制行程」视图（所有人可见）。

**操作步骤**：
1. 选择**旅行目的**（单选）。
2. 勾选**兴趣方向**（多选，最多 5 项，如美食 / 购物 / 文创 / 咖啡 / 历史 / 建筑 / 亲子等）。
3. 填写**出发日期**与**返程日期**。
4. （可选）勾选「**AI 增强生成（RAG）**」——见第 5 节说明。
5. 提交，系统生成逐日行程，包含：
   - 逐日行程（景点 / 店铺 / 活动）
   - 历史人物、建筑风貌
   - 美食购物推荐
   - 节事日历（仅展示与所选日期窗口重叠的活动）
   - 季节提示
   - **资料来源**（当启用 RAG 溯源时，见 3.1.4）

**降级说明**：若知识库不可达或未配置，系统会给出明确提示并回退到基于本地数据的**规则生成**，网站始终有内容、不会报错白屏（`FR-1` 约束③、`NFR-7`）。

#### 3.1.1 知识库状态提示（FR-8 / FR-9）
- 顶栏会展示**知识库状态**：正常为可用；若外部知识库不可达，显示「⚠ 知识库不可用」并仍可正常生成（降级）。
- 「**AI 增强生成**」选项的可用性取决于 RAG 配置：
  - 已配置密钥且索引就绪 → 可勾选。
  - 未配置或索引缺失 → 选项**置灰**，并提供「配置指引」入口（`FR-9`）。

#### 3.1.2 同步知识库（FR-7）
点击顶栏「**↻ 同步知识库**」按钮，可重新加载 WuDaDao 知识库；若已配置 RAG 密钥，会异步重建向量索引。同步状态会在按钮旁反馈。

#### 3.1.3 打印 / 导出（NFR-6）
行程页支持浏览器打印或导出 PDF；打印时表单与导航控件会自动隐藏，便于带走纸质攻略。

#### 3.1.4 行程溯源（FR-10）
当行程由 AI 增强（RAG）生成时，行程中会展示命中的**知识库来源**（`sources`），可点击**展开**查看来源标题、所属页面与关联度。规则生成时则展示"基于本地 data 与知识库的规则推荐"说明，不逐条溯源。

> ⚠️**待核对**：溯源区块的具体位置（行程底部或各条目内）与展开交互以线上为准。

### 3.2 游客 Wiki — 社区评价（FR-2 / FR-3 / FR-5 / FR-11 / FR-12）

**入口**：顶部「游客 Wiki」视图（所有人可见）。

#### 3.2.1 看评价（FR-3）
- 公开列表**仅展示管理员已通过（approved）的评价**，按"有用数降序 + 时间倒序"排列。
- 每条评价展示：评分星条、标题、正文、标签、评价对象、作者名、发布日期。
- 匿名评价显示「**匿名游客**」。
- 支持按**评价对象类型**过滤（整体行程 / 具体店铺 / 具体活动）。
- 已登录用户可看到并点击「**有用 👍 (n)**」与「**举报**」；匿名/游客仅可见计数。

#### 3.2.2 发评价（FR-2）
1. **先登录**（未登录提交将被拒绝）。
2. 在评价表单填写：
   - **评分**：1~5 星。
   - **评价对象**：整体行程 / 具体店铺 / 具体活动。
   - **标题**（可选）、**正文**（1~2000 字）、**标签**（可选）。
   - **匿名发布**：勾选后显示"匿名游客"，默认**实名**。
3. 提交后评价进入 **pending（待审核）** 状态，审核通过后才公开。

#### 3.2.3 我的评价（FR-5）
- **入口**：「我的评价」视图（仅登录用户可见）。
- 展示你发表的全部评价及**审核状态徽章**：⏳ 待审核 / ✅ 已通过 / ❌ 已拒绝。
- 被拒绝的评价会显示**审核意见**。

#### 3.2.4 有用投票（FR-11）
- 已登录用户对**已通过**评价点击「有用」即记录投票；**同用户不重复计票**。
- 再次点击可**取消**投票并扣减计数。
- 计数会参与公开列表排序，帮助好内容被看到。
- 匿名/游客**仅可见计数**，不可投票。

#### 3.2.5 举报（FR-12）
- 已登录用户可对**已通过**评价提交举报（含原因，如垃圾广告 / 不实信息 / 不当内容 / 骚扰 / 其他）。
- 原因为空、或同一用户对同一评价 **10 分钟内重复举报**会被拒绝。
- 举报**不向被举报者泄露举报人身份**。
- 提交后生成待处理工单进入管理员审核队列。

---

### 3.3 审核后台（管理员专用）（FR-4 / FR-13）

**入口**：「审核后台」视图（**仅 admin 角色可见**；未登录返回 401，非管理员返回 403）。

#### 3.3.1 评价审核（FR-4）
- 按状态筛选：**待审核 / 已通过 / 已拒绝 / 全部**。
- 操作：**通过** / **拒绝**；拒绝可填**审核意见**。
- 已通过的条目可改回拒绝，反之亦然。
- 所有审核操作记录**操作人与时间**。
- **批量审核**（FR-13）：勾选多条后批量通过/拒绝；单条失败不中断整批，汇总失败清单。

#### 3.3.2 举报处理（FR-13）
- 后台新增「**举报处理**」Tab，列出举报工单（富化显示被举报评价标题 / 状态 / 摘要 / 是否命中敏感词）。
- 处理决定：
  - **驳回举报（uphold）** → 评价维持 approved。
  - **采纳举报（dismiss）** → 评价转 rejected（从公开列表消失）+ 记录。
- 新评价进入待审/待处理时，对 `body`/`title` 做**敏感词预检**并标记高危，辅助治理。

## 4. 账号、隐私与安全（FR-6 / NFR-2 / NFR-4）

- **密码安全**：密码以 scrypt 加盐哈希存储，不存明文。
- **会话安全**：会话 token 存于 HttpOnly Cookie，跨站请求不携带（SameSite=Lax）。
- **匿名保护**：登录用户发评价可勾选匿名；举报人身份对普通用户保密。
- **数据保护**：`users.json` / `reviews.json` 已在 `.gitignore`，不入库。
- **默认管理员**：首次启动自动创建 `admin` / `admin12345`，请生产环境务必通过 `ADMIN_USER` / `ADMIN_PASS` 修改（`DESIGN.md` §8.2）。

## 5. AI 增强（RAG）说明（FR-8 / FR-9 / FR-10）

**什么是 AI 增强**：在 WuDaDao 知识库之上，用向量检索 + 大模型生成更贴合你输入的行程，并给出"依据哪条知识"的溯源。

**何时可用**：
- 部署方在 `.env` 配置了 `RAG_API_KEY` 且知识库索引就绪时，「AI 增强生成」选项可勾选。
- 未配置时，选项**置灰**并给配置指引；行程仍可用**规则生成**。

**会失败吗**：若勾选后 RAG 实际生成失败，系统自动**降级为规则生成**并提示"已切换为常规生成"，**不会返回 500 错误**（`FR-9` Unwanted 规则）。

> 普通用户无需理解技术细节；只需知道：勾了"AI 增强"体验更个性、可溯源；没勾或不可用也能正常出行程。

## 6. 常见问题（FAQ）

**Q1：为什么我点"发表评价"没反应 / 被拒绝？**
A：发表评价**必须登录**（依据 PO 决策 REG-14）。请先注册/登录；登录后默认实名，可勾选匿名发布。

**Q2：为什么"AI 增强生成"是灰的不能点？**
A：说明部署方未配置 RAG 密钥或索引缺失（FR-9）。点击会有配置指引；不影响正常规则生成行程。可联系部署/运维配置 `RAG_API_KEY`。

**Q3：顶栏显示"⚠ 知识库不可用"是什么意思？**
A：外部 WuDaDao 知识库路径未配置或不可达（FR-8/NFR-7）。网站会明确降级、仍可基于本地数据生成行程，不会白屏。可联系运维检查 `WUDADAO_KB_PATH`。

**Q4：我发的评价为什么别人看不到？**
A：所有评价需**管理员审核通过**才公开（NFR-3）。可在「我的评价」查看状态：⏳ 待审核 / ✅ 已通过 / ❌ 已拒绝（被拒会显示意见）。

**Q5：点"有用"计数为什么没加？**
A：需**登录**且评价为**已通过**状态；同一用户不重复计票，再次点击为取消。

**Q6：举报后会暴露是我举报的吗？**
A：不会。举报人身份仅管理员可见，不向被举报者泄露（FR-12）。

**Q7：刷新/重启后我要重新登录吗？**
A：当前会话为内存态，服务重启需重新登录（DESIGN.md §8.3）。

**Q8：行程怎么打印带走？**
A：用浏览器打印/导出 PDF 即可，打印时表单与导航会自动隐藏（NFR-6）。

> 提示：若某问题**高频出现**，建议将其整理为 FAQ 沉淀到团队知识库（游客 Wiki / 帮助中心），并关联本手册对应章节。

## 7. 运维附录（部署与管理员参考）

> 本节面向部署方/管理员，普通游客可跳过。

### 7.1 部署
```bash
cd app
npm start          # 默认 http://localhost:3000
```
运行环境：Node.js >= 18。

### 7.2 环境变量（`.env`，已被 gitignore；参考 `.env.example`）
| 变量 | 说明 | 默认 |
| --- | --- | --- |
| `PORT` | 监听端口 | `3000` |
| `ADMIN_USER` / `ADMIN_PASS` | 管理员账号 | `admin` / `admin12345` |
| `WUDADAO_KB_PATH` | 外部知识库目录（绝对路径或 `file://` URL），留空回退仓库内默认 `WuDaDao/` | 空 |
| `RAG_API_KEY` | RAG 服务商密钥，未配置则全程降级规则生成 | 空 |
| `RAG_BASE_URL` | API Base（默认智谱） | 智谱 |
| `RAG_EMBED_MODEL` / `RAG_LLM_MODEL` | 向量/对话模型 | `embedding-3` / `glm-4-flash` |
| `ANALYTICS_ENDPOINT` | 埋点上报端点，未配置则静默 no-op | 空 |

### 7.3 RAG 建索引
```bash
npm run rag:index     # 或 node app/rag/index.js，依据 WUDADAO_KB_PATH 建 app/data/rag-index.json
```

### 7.4 生产注意
- 内存会话：多实例需 Redis/DB；建议前置反向代理（Nginx/Caddy）。
- 密码哈希参数（scrypt N/r/p）生产可调高。
- 远程 HTTP(S) 知识库拉取暂不支持（FR-8 仅本地目录 / `file://`）。
