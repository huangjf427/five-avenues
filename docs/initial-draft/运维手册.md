## 1. 概述

FiveAvenues 是一个 **零 npm 依赖** 的 Node.js 原生 HTTP 服务，为天津五大道景区提供「个性化行程生成 + 社区评价」。

| 项 | 说明 |
| --- | --- |
| 运行环境 | **Node.js >= 18** |
| 第三方依赖 | **无**（仅用内置 `http/fs/crypto/path/url`） |
| 启动命令 | `cd app && npm start` |
| 默认端口 | `3000`（`PORT` 可改） |
| 会话机制 | 进程内内存 token（Cookie `fa_session`，7 天） |
| 外部依赖 | 可选：WuDaDao 知识库目录、RAG 服务商（智谱/DeepSeek/OpenAI 兼容） |

> 来源：`DESIGN.md` §1、§8。

## 3. 安装与启动

```bash
# 1. 进入应用目录
cd app

# 2. 配置环境变量（见第 4 节）
cp ../.env.example .env      # 或直接放到 app/.env
vim .env                     # 填入实际值

# 3. 建 RAG 索引（仅当启用 AI 增强时需要）
npm run rag:index            # 生成 app/data/rag-index.json（依据 WUDADAO_KB_PATH）

# 4. 启动
npm start
# 访问 http://localhost:3000
```

> `npm start` 实际为 `node server.js`；仓库零依赖，无需 `npm install`（`DESIGN.md` §8.1）。

## 4. 配置（`.env`）

复制 `.env.example` 为 `.env`（`.env` 已被忽略，不会入库）。变量说明：

| 变量 | 说明 | 默认 | 来源 |
| --- | --- | --- | --- |
| `PORT` | 监听端口 | `3000` | `DESIGN.md` §8.2 |
| `ADMIN_USER` | 管理员用户名 | `admin` | §6 |
| `ADMIN_PASS` | 管理员密码 | `admin12345` | §6 |
| `WUDADAO_KB_PATH` | 外部知识库目录（FR-8）：绝对路径或 `file://` URL；留空回退仓库内默认 `WuDaDao/` | 空 | `DESIGN.md` §8.2 / `overview-fr8.md` |
| `RAG_API_KEY` | RAG 服务商密钥；**未配置则全程降级规则生成** | 空 | `DESIGN.md` §9.3 |
| `RAG_BASE_URL` | API Base（智谱 `https://open.bigmodel.cn/api/paas/v4`） | 智谱 | §9.3 |
| `RAG_EMBED_PATH` / `RAG_CHAT_PATH` | 非标准网关路径覆盖 | `/embeddings` / `/chat/completions` | `.env.example` |
| `RAG_EMBED_MODEL` / `RAG_LLM_MODEL` | 向量 / 对话模型 | `embedding-3` / `glm-4-flash` | §9.3 |
| `ANALYTICS_ENDPOINT` | 埋点上报端点（NFR-8）；**未配置则 `/api/analytics` 静默 no-op** | 空 | `DESIGN.md` §3.2 / `REQUIREMENTS.md` NFR-8 |

> 解析方式：自写 `loadDotEnv`，不引入 dotenv（零依赖约定，`DESIGN.md` §1 / `.workbuddy` 约定）。

## 5. 知识库配置与同步（FR-7 / FR-8 / NFR-7）

- **配置化接入**：WuDaDao 知识库位置**不得硬编码**，必须通过 `WUDADAO_KB_PATH` 配置（`REQUIREMENTS.md` FR-8 / NFR-7）。
- **可达性降级**：
  - 未配置 → 回退仓库内默认 `WuDaDao/`（开箱即用）。
  - 目录不存在 / 无读权限 → 行程生成降级为本地 `data/*.json` 规则生成，`/api/meta` 返回 `wikiStatus:"degraded"` 与错误原因，前端顶栏显示「⚠ 知识库不可用」，但**不返回 500、不静默失败**。
- **同步**：界面顶栏「**↻ 同步知识库**」按钮重新加载知识库；若已配置 `RAG_API_KEY`，会异步重建向量索引（返回 `ragRebuild:"started"`）。
- **远程拉取**：FR-8 本版**仅支持本地目录 / `file://`**，远程 HTTP(S) 拉取留待后续（`overview-fr8.md` §五）。

## 6. 账号与权限（FR-4 / FR-6 / NFR-4）

- **默认管理员**：首次启动自动创建 `admin` / `admin12345`。**生产务必通过 `ADMIN_USER` / `ADMIN_PASS` 修改**（见 §4）。
- **角色**：`visitor` / `admin`。管理员接口分级鉴权：未登录 `401`、非管理员 `403`。
- **密码安全**：scrypt 加盐哈希（N=16384, r=8, p=1），每用户独立 salt，不存明文（`DESIGN.md` §5）。
- **账号文件**：`app/data/users.json` 自动生成，**已 gitignore**，请勿提交。

---

## 7. 安全要点（NFR-4 / DESIGN §5）

| 措施 | 说明 |
| --- | --- |
| 密码哈希 | scrypt（Node 内置），生产可调高 N/r/p |
| 会话隔离 | HttpOnly + SameSite=Lax Cookie，跨站不携带 |
| 内容清洗 | 评价 `body/title` HTML 转义（防 XSS） |
| 数据保护 | `users.json` / `reviews.json` / `reports.json` 在 `.gitignore` |
| 匿名保护 | 评价可匿名；举报人身份仅管理员可见 |

## 8. 本地数据源维护（`app/data/`）

数据文件为结构化 JSON，可手动编辑扩充，修改后点「↻ 同步」或重启生效（`app/data/README.md`）：

| 文件 | 作用 | 备注 |
| --- | --- | --- |
| `shops.json` | 商业店铺（餐饮/咖啡/文创/购物/书店/甜品） | 字段含 `id/name/category/address/road/features/tags/summary/rating/relatedPages` |
| `activities.json` | 本地文化/旅游活动 | 仅与所选日期窗口重叠时出现在「节事日历」 |
| `feedback.json` | 游客反馈（**仅排序加权，不展示**） | 平均评分 ≥4.5 ×1.2、≥4.0 ×1.1、<3.0 ×0.9 |
| `reviews.json` | 游客 Wiki 评价（**面向公众，需审核**） | 仅 `approved` 展示 |
| `users.json` | 账号（自动生成） | 含密码哈希，勿入库 |

**编辑约定**：
- 保持合法 JSON（双引号、无尾逗号）；新增条目补全唯一 `id` 与 `tags`。
- 兴趣标签需与 `src/recommender.js` 的 `INTERESTS` 对应；新增兴趣须同步改该文件。
- 文件缺失 / JSON 解析失败 → 自动降级为空，不影响主流程。

## 9. RAG（AI 增强）运维（FR-8~FR-10 / DESIGN §9）

- **建索引**：`npm run rag:index`（或 `node app/rag/index.js`），依据 `WUDADAO_KB_PATH` 生成 `app/data/rag-index.json`。
- **运行期**：启动时空索引后台补建；「↻ 同步」依据 `.env` 异步重建。
- **降级链**：未配 `RAG_API_KEY` / 索引缺失 / 模型不可用 / 输出非法 → 自动回退规则生成，前端可感知（选项置灰 + 配置指引 + 溯源徽章），**永不 500**。
- **服务商兼容**：OpenAI 兼容网关，支持智谱/DeepSeek/OpenAI，通过 `RAG_BASE_URL` / `RAG_EMBED_MODEL` / `RAG_LLM_MODEL` 覆盖。
- **已知**：真实端到端（有效密钥 + 真实 embedding）需在生产/有效密钥环境验证（环境密钥无效时仅验证降级链路，`overview-fr8.md` §五）。

## 10. 日志、监控与埋点（NFR-8）

- **服务日志**：`app/server.log`（启动日志、知识库来源、RAG 重建状态）。
- **埋点**：`/api/analytics` 接收端，经 `ANALYTICS_ENDPOINT` 转发；**未配置则 no-op，失败静默**，不引入依赖、不影响主流程。
- **上报字段合规**（NFR-8）：仅计数/标签/原因分类/长度；不采集密码明文与评价正文全文；匿名/游客不关联用户标识。
- **建议**：确认 `ANALYTICS_ENDPOINT` 后，按 `docs/PRD_V1.2` §7.3 建立北极星（周生成成功用户数）、RAG 使用率 vs 降级率、评价发布/通过/互动/举报时效看板。

## 12. 升级与回滚

- **升级**：拉取新版本 → 更新 `.env`（如有新增变量，参考对应 `.env.example`）→ 如需 RAG 则重建索引（`npm run rag:index`）→ 重启。
- **回滚**：保留上一版本 tag/commit；回退代码并重启即可（数据文件向后兼容时无需回退）。
- **配置漂移检查**：升级后对比 `DESIGN.md` / `REQUIREMENTS.md` 的环境变量表，确认 `.env` 字段完整。

## 13. 生产部署建议（DESIGN §8.3）

- **反向代理**：静态资源与 TLS 建议前置 Nginx/Caddy，将 `3000` 暴露在反代之后。
- **会话持久化**：当前为内存会话，**多实例需改用 Redis 或数据库**；否则多副本会话不共享、重启需重新登录。
- **密码参数**：生产调高 scrypt N/r/p。
- **密钥管理**：`.env` 不入库；用密钥管理服务或部署平台环境变量注入，避免明文落在镜像。
- **监控**：关注 `server.log` 中 `wikiStatus` / `ragRebuild` 状态，异常时告警。

## 14. 故障排查（排错表）

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 顶栏「⚠ 知识库不可用」 | `WUDADAO_KB_PATH` 未配置或路径不可达 | 检查路径/权限；留空则回退默认 `WuDaDao/` |
| 行程生成慢/无来源 | 未配 `RAG_API_KEY` 或索引缺失 | 配置密钥并执行 `npm run rag:index`；否则走规则生成 |
| 「AI 增强生成」置灰 | RAG 未配置/索引缺失 | 配置 `RAG_*` 并重建索引；用户侧仅影响增强项 |
| 提交评价被拒（401） | 未登录 | 先注册/登录（NFR-2 / REG-14） |
| 评价不公开 | 状态为 pending/被拒 | 管理员审核通过；被拒见审核意见 |
| 重启后需重新登录 | 内存会话 | 预期行为；生产改用持久化会话 |
| 埋点无数据 | `ANALYTICS_ENDPOINT` 未配置 | 配置端点；未配置为静默 no-op |
| 远程知识库拉取失败 | FR-8 不支持远程 HTTP(S) | 本版仅本地目录 / `file://`，见 `overview-fr8.md` |
