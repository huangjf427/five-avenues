## 一、调研背景与目标
FiveAvenues 是面向**天津五大道景区**的文旅网站：用 RAG 读取本地文史知识库（WuDaDao）生成**个性化 AI 行程**，并提供**审核制匿名 UGC 评价社区**；技术栈零 npm 依赖、可自托管、隐私友好、免费、不含交易闭环。已完成 V1.0–V1.1，V1.2 规划 RAG 增强与社区运营增强。

**目标**：在"游客规划行程 + 参考他人评价"场景下，界定竞品范围、量化能力差距、提炼护城河与机会，为 V1.2 优先级与定位/变现决策提供依据。

## 二、方法说明
- 来源：仓库 `main` 分支需求/设计文档（REQUIREMENTS.md、DESIGN.md、overview-fr8.md）+ 4 集群并行联网调研（WebSearch/WebFetch，带来源与时间）。
- 竞品分层：直接/间接替代、地图+AI 行程、通用大模型、国际对标、官方互补。
- 原则：**事实 / 判断 / 建议分离**；不确定数据已标注。

## 三、核心发现
1. **竞争分三层**：UGC 点评（点评/小红书/马蜂窝）、地图+AI 行程（高德/百度/携程）、通用大模型；官方为互补方。
2. **FA 唯一组合**：垂直 RAG 文史行程 + 审核匿名 UGC + 零依赖自托管 + 隐私友好——**竞品均无此组合**。
3. **巨头盲区**：点评/携程赌"广度+交易"，官方赌"权威+体验"，**无人赌"垂直可信本地知识 + 隐私友好 + 可自托管"**。
4. **最大威胁**：高德"小高老师"、百度 AI 向导、携程 AI 行程、豆包等**免费 AI 行程**截流"普通规划"需求，FA 须以"无幻觉可信 + 本地深度"守细分。
5. **官方是最佳盟友**：权威史料可作 RAG 知识源，且官方不做个性化行程与 UGC 社区。

## 五、竞品差距地图（FA 应对策略）
- **vs 点评/小红书**：我们有本地深度行程 + 匿名可信社区；缺海量 UGC/流量 → **互补 + 内容导入，不硬刚**。
- **vs 地图/携程**：我们有垂直文史 RAG + 隐私；缺导航/交易 → **互补（导出/嵌入），不自建交易**。
- **vs 通用大模型**：我们有无幻觉可信 + 本地 RAG；缺广度 → **强调专业度与可溯源**。
- **vs 官方**：我们有个性化行程 + UGC；缺权威 → **接入官方作知识源 + 争取背书**。

## 六、SWOT
- **S**：垂直街区 RAG 深度；隐私友好（零依赖/scrypt/匿名/埋点合规）；可自托管零成本；RAG 溯源+规则降级防幻觉；审核匿名 UGC 风险可控。
- **W**：无交易闭环变现弱；无地图导航行中断层；UGC 冷启动；单景区天花板；AI 依赖外部 API；零品牌流量获客难。
- **O**：官方史料接入 RAG + 合作背书；V1.2 社区质量建可信 UGC 壁垒；隐私合规叙事；文保/研学自托管版（B2G）；巨头盲区。
- **T**：巨头 AI 免费截流；点评/小红书满足看评价；官方 AR 导览；大模型联网生成行程；单景区 C 端变现难。

## 八、风险提示
- **R1 巨头 AI 截流**：免费 AI 行程可能让普通规划需求不流向 FA。
- **R2 UGC 冷启动**：新站评价稀缺，社区壁垒短期难建。
- **R3 单景区天花板 + C 端变现难**：需探索 B2G/B2B（自托管/定制知识库）。
- **R4 RAG 真实效果未验证**：需求文档注明环境未做端到端验证，需真实数据背书。

## 九、建议动作
**给产品团队**
1. 优先落地 V1.2 社区质量（FR-11/12/13）——可信 UGC 是差异核心。
2. 做 RAG 溯源与引导（FR-9/10），显性化"可信生成"。
3. 新增"行程导出/分享到高德/小红书"——低成本导航补位。
4. 推动接入官方史料作 RAG 知识源。

**给管理层**
1. 定位"**垂直可信本地知识 + 隐私友好**"细分，不与巨头正面对抗。
2. 官方/文保合作是获客与背书关键路径，建议设专项。
3. 商业模式：免费工具 + 政企**自托管/定制知识库服务（B2G/B2B）**可能优于 C 端广告。
4. 重点防范 R1–R4，用真实 RAG 评测与用户访谈验证核心假设。
